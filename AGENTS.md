# AGENTS.md

This file records Codex execution rules for Alex's personal portfolio website project.

## Project Purpose

Build and maintain a bilingual personal portfolio website for Alex / 阿涛.

The site presents selected projects, media and communication practice, AI workflow experiments, and lightweight GPT-to-Codex prototypes.

## Core Rules

- Keep the site bilingual with separate routes: `/en` and `/zh`.
- Do not mix English and Chinese paragraph-by-paragraph on the same page.
- Keep English and Chinese versions visually consistent, while localizing copy for each audience.
- Preserve the dark editorial visual direction: black background, oversized white typography, black-and-white portrait, restrained pink accent, high contrast, and minimal motion.
- Prioritize recruiter readability and clear project evidence over visual gimmicks.
- Do not change approved project names, page structure, or core positioning unless explicitly requested.
- Do not add new social links unless explicitly requested.
- Do not alter the portrait with AI or change facial features, expression, hairstyle, clothing, or lighting.
- Use only CSS-style portrait treatment such as grayscale, dark overlay, opacity, contrast, and gradient mask.
- Keep interactions lightweight and avoid heavy scroll hijacking, game-like effects, or complex motion graphics.

## Technical Direction

- Preferred stack: Next.js, TypeScript, and CSS/Tailwind-style utility-friendly structure.
- Keep components reusable and easy to update.
- Keep assets in `public/media` or appropriate public asset folders.
- Use real project screenshots, videos, and documents when available.
- Use consistent dark mockup frames for project screenshots and demos.
- Run a local build or equivalent check after meaningful code changes when feasible.

## Do Not Modify Without Request

- Hero positioning and portrait treatment
- Selected Projects structure
- Project detail page structure
- Experience entries
- Footer links
- Existing media assets and project evidence
